### Question 1

What problem does Docker primarily solve?

- A) Making applications run faster
- B) Replacing virtual machines entirely
- C) Ensuring consistent environments across development, testing, and production
- D) Providing a graphical interface for server management

<details>
<summary>Show Answer</summary>

**C) Ensuring consistent environments across development, testing, and production**

Docker solves the "works on my machine" problem by packaging applications with their entire environment — OS libraries, dependencies, and configuration — into portable containers.

</details>

---

### Question 2

What is the relationship between a Docker image and a Docker container?

- A) They are the same thing
- B) An image is a running container
- C) A container is a running instance of an image
- D) A container is a template for creating images

<details>
<summary>Show Answer</summary>

**C) A container is a running instance of an image**

An image is a read-only template (like a class in programming). A container is a running instance of that image (like an object). One image can create many containers.

</details>

---

### Question 3

Which of these is NOT a difference between containers and virtual machines?

- A) Containers share the host OS kernel; VMs have their own OS
- B) Containers start in seconds; VMs start in minutes
- C) Containers are measured in MBs; VMs in GBs
- D) Containers cannot run on Linux; VMs can

<details>
<summary>Show Answer</summary>

**D) Containers cannot run on Linux; VMs can**

Containers run on Linux, macOS, and Windows. In fact, containers originated on Linux and are most commonly used there. All other statements are true differences.

</details>

---

### Question 4

What is a Dockerfile?

- A) A running Docker container
- B) A text file with instructions to build a Docker image
- C) A configuration file for Docker Desktop
- D) A log file generated by Docker

<details>
<summary>Show Answer</summary>

**B) A text file with instructions to build a Docker image**

A Dockerfile is like a recipe — it contains step-by-step instructions (FROM, COPY, RUN, etc.) that tell Docker how to build an image.

</details>

---

### Question 5

What is Docker Hub?

- A) A local tool for managing containers
- B) A command-line interface for Docker
- C) A public registry where Docker images are stored and shared
- D) A networking protocol used by containers

<details>
<summary>Show Answer</summary>

**C) A public registry where Docker images are stored and shared**

Docker Hub is the default public registry, like an "app store" for Docker images. You can push your images to it and pull images others have shared.

</details>
